為什麼要學變數？

可將常需要用到的CSS屬性值放在變數來管理,常會輸入到5~10次的樣式就可以使用變數。

變數教學

  • 可將常需要設定的CSS放在變數來管理
  • 變數是用美元符號開頭$

$變數名稱: 值;

ex:

$text-color: #8f3333;

$font-m: 16px;

常見變數編譯錯誤

  1. 忘記加分號 $變數名稱: 值;
  1. 變數沒有放在最前面 在編譯過程中,是從上到下來執行,所以變數要寫在最前面,不然在編譯過程中會因為找不到變數而會出錯。

變數搭配加減乘除好方便

  • SCSS有支援加減乘除,所以可在變數使用

$變數名稱: 值;

ex:

$font-l: $font-m * 1.2;

$font-m: 16px;

$font-s: $font-m * 0.8;

補充: 顏色的變數在命名上我們會盡可能避免使用顏色來命名，其主要原因是因為當顏色若修改就會變得很奇怪，例如 $red 後來色碼改成黑色，那麼就會顯得相當奇怪。變數在命名上可以試著參考較知名的框架命名,像是Bootstrap。

駝峰式大小寫（Camel-Case）

  • 小駝峰式命名法（lower camel case）：第一個單字以小寫字母開始；第二個單字的首字母大寫，例如：firstName、lastName。CSS常用此命名法。
  • 大駝峰式命名法（upper camel case）：每一個單字的首字母都採用大寫字母，例如：FirstName、LastName、CamelCase，也被稱為Pascal命名法。在其他程式語言為了產生實例或是類別才會使用此命名法。

拋棄語意命名

拋棄語意命名方式,改以工具、功能性質來命名

架構: 拋棄直白式命名方式,改以工具性質來命名。ex: .bookList{} 及.bookList-title{}跟.bookList-img{},改成.list{}及.list-title{}跟.list-img{}, 就算用在其他區塊如DVD或是其他產品列表也不會因為Class名稱而感到突兀

CSS 範圍性常見錯誤

在很多地方下相同的CSS樣式設定。ex .header p{ font-size: 16px } .content p{ font-size: 16px }

可以改在最外圍用.wrap或是.container包圍,class具有範圍性或是在body,html設定。ex .wrap{ font-size: 16px } body,html { font-size: 16px }。

若是有其他文字大小的設定,再用class選擇器覆蓋過去

請勿將高度寫死

使用padding或是margin來做適當的留白,在響應式下,會自適應調整伸縮。

請勿使用 ID 綁定網頁樣式

id通常是做為錨點,用來切換位置。一個網頁上只能有1個id,無法像class重覆使用。此外,id的權重分數是100分而class的權重分數是10分,使用id選擇器很難去覆蓋

解決 JS 與 CSS 功能相互干擾

使用js前缀詞,以避免class名稱改變時,使得以class名稱所綁定的JS功能失效。在Vue.js、Angular等前端框架是不會這樣寫,但在jQuery及JS原生寫法沒有用到任何框架,必須注意到這個細節。
